# Web Accessibility (A11Y)

## Overview

In this project, I explored the principles of **web accessibility** and how to design websites that can be used by as many people as possible, including those relying on assistive technologies. The goal was to understand how accessible interfaces improve usability for everyone and not only for users with disabilities.

## What I practiced

During this project, I focused on improving the structure and semantics of web pages so they can be interpreted correctly by screen readers and other assistive tools. I also paid attention to navigation, readability, and visual contrast to ensure that content remains accessible under different conditions.

This work involved applying accessibility guidelines and understanding how browsers and assistive technologies interact with HTML and CSS.

## Concepts covered

Through this project I learned and practiced:

- The purpose of **ARIA attributes** and how they help assistive technologies understand interface components.
- The different **WCAG conformance levels** (A, AA, AAA) and how they define accessibility standards.
- Why **web accessibility is essential** for inclusive design and better user experiences.
- How to test accessibility using various **tools and browser features**.
- Good practices for building interfaces that remain usable without a mouse or with assistive technologies.

## Goal

The objective of this project was to better understand how accessibility fits into modern web development and how thoughtful design choices can make digital content available to a wider audience while maintaining clean and structured code.
