# Responsive Web Design

## Overview

In this project, I explored the fundamental principles of **responsive web design** and how to build web pages that adapt smoothly to different screen sizes and devices. The goal was to understand how modern websites adjust their layout and content to provide a consistent and usable experience on desktops, tablets, and mobile devices.

## What I practiced

During this project, I focused on designing layouts that remain flexible and readable regardless of the device being used. I experimented with techniques that allow elements to scale, reposition, or reorganize themselves depending on the available screen space.

This work involved structuring HTML with responsiveness in mind and using CSS features that allow layouts to adapt dynamically.

## Concepts covered

Through this project I learned and practiced:

- The **mobile-first approach**, where the design starts with small screens and progressively adapts to larger ones.
- The use of **media queries** to apply different styles depending on screen size or device characteristics.
- Choosing appropriate **breakpoints** to maintain readable and balanced layouts.
- Using flexible **CSS units** such as `rem`, `%`, `vw`, and `vh` to make elements scale naturally.
- The difference between **responsive design** (fluid layouts that adapt continuously) and **adaptive design** (layouts designed for specific screen sizes).

## Goal

The main objective of this project was to understand how to design interfaces that remain usable and visually consistent across a wide range of devices while keeping the code simple, maintainable, and scalable.

By applying these principles, it becomes possible to build modern websites that provide a good user experience regardless of screen size.
